E290887 - ARCHIVO: VBRun60sp5.exe instala archivos de tiempo de ejecución del SP5 de Visual Basic 6.0 

 


                  ARCHIVO: VBRun60sp5.exe instala archivos de tiempo de 
                  ejecución del SP5 de Visual Basic 6.0

              La información en este artículo se refiere a: 

              - Microsoft Visual Basic Learning Edition para Windows 6.0 
              - Microsoft Visual Basic Professional Edition para Windows 6.0 
              - Microsoft Visual Basic Enterprise Edition para Windows 6.0 


            Resumen
            VBRun60sp5.exe es un archivo ejecutable autoextraíble que instala 
            las versiones más recientes de los archivos de tiempo de ejecución 
            de Microsoft Visual Basic, que todas las aplicaciones creadas con 
            Visual Basic 6.0 necesitan. Los archivos incluyen las revisiones 
            incluidas en el Service Pack 5 de Visual Studio 6.0. 


            Más Información
            El archivo siguiente se puede descargar desde el Centro de descarga 
            de Microsoft:

              VBRun60sp5.exe 
            Fecha de publicación: 27.02.01

            Para obtener información adicional acerca de cómo descargar archivos 
            de soporte técnico de Microsoft, haga clic en el número de artículo 
            siguiente para verlo en Microsoft Knowledge Base: 
              E119591 Cómo obtener archivos de soporte técnico de Microsoft 
              desde los servicios en línea 
            Microsoft ha utilizado el software de detección de virus más 
            reciente en el momento de su publicación para buscar virus en este 
            archivo. Una vez publicado, el archivo se encuentra en servidores 
            seguros que impiden cambios no autorizados en el archivo.

            ¿Necesito el archivo VBRun60sp5.exe? 
            No se pretende que VBRun60sp5.exe reemplace al Asistente de 
            empaquetado y distribución (PDW) para distribuir las aplicaciones de 
            Visual Basic. Por ejemplo, si la aplicación incluye componentes como 
            controles ActiveX o archivos DLL, debe usar el PDW o un paquete de 
            instalación de terceros para distribuir las aplicaciones. Sin 
            embargo, si la aplicación de Visual Basic sólo depende de los 
            archivos incluidos en el archivo VBRun60sp5.exe, basta con 
            proporcionar a los usuarios finales el archivo ejecutable (.exe) y 
            VBRun60sp4.exe para distribuir la aplicación. 

            Para determinar si la aplicación requiere archivos adicionales para 
            su distribución, puede usar el PDW para crear un conjunto de 
            archivos de instalación de prueba. Cuando el PDW crea un paquete de 
            instalación, genera también un archivo Setup.lst. Puede abrir el 
            archivo Setup.lst en cualquier editor de textos, como el Bloc de 
            notas. Si sólo aparece su archivo .exe en la sección [Setup1 Files] 
            del archivo Setup.lst resultante, los usuarios finales podrán 
            ejecutar la aplicación después de ejecutar VBRun60sp5.exe para 
            instalar los archivos de tiempo de ejecución esenciales. Sin 
            embargo, si aparecen varios archivos en la sección [Setup1 Files], 
            debe considerar la posibilidad de utilizar el PDW o un paquete de 
            instalación de terceros para distribuir la aplicación. 

            Además de distribuir archivos ejecutables sencillos, también puede 
            utilizar el archivo VBRun60sp5.exe para lo siguiente: 
              Minimizar el tamaño de las descargas de aplicaciones de Visual 
              Basic desde Internet. 

              Si ejecuta el archivo Vbrun60sp5.exe por adelantado, los usuarios 
              pueden descargar su aplicación más rápidamente desde el Web. 


              Como método de solución de problemas cuando se produce un error en 
              la instalación de Visual Basic o una aplicació³n de Visual Basic. 

              Si durante la instalación aparece un mensaje de error que hace 
              referencia a uno de los archivos principales, o si se produce un 
              error durante la instalación al registrar un componente, es 
              posible que no haya coincidencia con los archivos principales del 
              equipo de destino. Si las versiones de los archivos incluidos en 
              VBRun60sp5.exe son más recientes que las versiones existentes en 
              el equipo de destino, puede resolver el problema si ejecuta el 
              archivo VBRun60sp5.exe antes de ejecutar el programa de 
              instalación.

              Q194754 SETUP.EXE File Linked to Missing Export OLEAUT32.DLL:185 
            Puede obtener un archivo ejecutable autoextraíble que instala los 
            archivos de tiempo de ejecución de Visual Basic 6.0. Para ello, 
            extrae el archivo VBRun60.exe del archivo VBRun60.cab, que se 
            encuentra disponible en los CD de Visual Basic y Visual Studio. El 
            archivo VBRun60.cab se encuentra en la carpeta 
            \Common\Tools\Vb\Cabinets del CD de Visual Basic 6.0 (Disco 1). 

            En los CD de Visual Studio 6.0 Professional, el archivo VBRun60.cab 
            está ubicado en la misma ruta relativa, pero en el disco 2. En los 
            CD de Visual Studio 6.0 Enterprise, se encuentra en el disco 3. 

            NOTA: la versión del archivo VBRun60sp5.exe que se incluye en este 
            artículo de Knowledge Base es distinta de la que aparece en el CD de 
            instalación. La versión del artículo es la más reciente que se 
            incluye en el Service Pack 5 de Visual Studio 6.0. 

            Puede emplear la utilidad Extract para extraer el archivo 
            VBRun60.exe del archivo contenedor VBRun60.cab. Puede encontrar 
            Extract.exe en la carpeta Windows o en el CD de instalación de 
            Microsoft Windows 95, Windows 98 o Windows NT. Desde el símbolo del 
            sistema de MS-DOS puede ejecutar el siguiente comando para extraer 
            el archivo desde el archivo contenedor al directorio actual: 
              Extract Vbrun60.cab 
            Para obtener información adicional acerca del uso de la utilidad 
            Extract, haga clic en el número de artículo siguiente para verlo en 
            Microsoft Knowledge Base: 
              E129605 Cómo extraer los archivos de Windows comprimidos 
              originales
